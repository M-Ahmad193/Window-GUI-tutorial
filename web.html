Windows Programming Basics

Learn the fundamental concepts of Windows GUI programming and how to create your first window application.

Understanding Windows Architecture

Windows GUI programming is based on the Windows API (Application Programming Interface), which provides functions for creating windows, handling user input, and drawing graphics. Before diving into code, let's understand the key components:

  • Window: A rectangular area on the screen where an application displays output and receives input
  • Message Loop: A mechanism that retrieves and dispatches messages to the appropriate window procedure
  • Window Procedure: A function that processes messages sent to a window
  • Device Context: A structure that defines a set of graphic objects and their attributes

Windows Architecture Overview

How It Works

When you create a Windows application, the operating system sends messages to your application when events occur (like mouse clicks or key presses). Your application processes these messages in the message loop and responds accordingly.

  • System detects user action
  • System creates a message
  • Message is placed in application queue
  • Application retrieves and processes message
  • Application performs appropriate action

Essential Functions

Let's explore the core functions you'll need to create a basic Windows application:

RegisterClassEx()

Registers a window class for subsequent use in calls to the CreateWindow or CreateWindowEx function.

Why Use It?

Before creating a window, you must register a window class that defines the window's attributes and behavior.

Essential for both Matrix and Encryption projects to set up the basic window structure.

Parameters
  • lpwcx: Pointer to a WNDCLASSEX structure
  • cbSize: Size of the structure
  • style: Class styles
  • lpfnWndProc: Pointer to window procedure
  • hInstance: Handle to application instance
  • hIcon: Handle to class icon
  • hCursor: Handle to class cursor
  • hbrBackground: Handle to class background brush
  • lpszClassName: Pointer to class name string

CreateWindowEx()

Creates an overlapped, pop-up, or child window with an extended window style.

Why Use It?

This is the function that actually creates the window on the screen. You'll use it to create the main window for your projects.

For Matrix project: Create a window with grid layout for matrix input and output.

For Encryption project: Create a window with text areas and encryption controls.

Parameters
  • dwExStyle: Extended window style
  • lpClassName: Registered class name
  • lpWindowName: Window title
  • dwStyle: Window style
  • x, y: Initial horizontal and vertical position
  • nWidth, nHeight: Window width and height
  • hWndParent: Handle to parent window
  • hMenu: Handle to menu or child identifier
  • hInstance: Handle to application instance
  • lpParam: Additional application data

ShowWindow()

Sets the specified window's show state.

Why Use It?

After creating a window, you need to make it visible on the screen. This function controls how the window is displayed.

Essential for both projects to display the application window.

Parameters
  • hWnd: Handle to window
  • nCmdShow: Show state (SW_SHOW, SW_HIDE, etc.)

UpdateWindow()

Updates the client area of the specified window by sending a WM_PAINT message to the window.

Why Use It?

This function forces the window to repaint its client area immediately. Useful when you need to update the display.

For Matrix project: Update display when matrix operations are performed.

For Encryption project: Update display when text is encrypted/decrypted.

Parameters
  • hWnd: Handle to window

Creating Your First Window

Now let's put these functions together to create a basic window application:

Basic Window Code

#include <windows.h>

// Window Procedure function
LRESULT CALLBACK WindowProc(HWND hwnd, UINT uMsg, WPARAM wParam, LPARAM lParam);

int WINAPI WinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance, LPSTR lpCmdLine, int nCmdShow) {
    // Register the window class
    const wchar_t CLASS_NAME[] = L"MyWindowClass";
    
    WNDCLASSEX wc = {};
    wc.cbSize = sizeof(WNDCLASSEX);
    wc.style = CS_HREDRAW | CS_VREDRAW;
    wc.lpfnWndProc = WindowProc;
    wc.hInstance = hInstance;
    wc.lpszClassName = CLASS_NAME;
    wc.hbrBackground = (HBRUSH)(COLOR_WINDOW + 1);
    wc.hCursor = LoadCursor(NULL, IDC_ARROW);
    wc.hIcon = LoadIcon(NULL, IDI_APPLICATION);
    wc.hIconSm = LoadIcon(NULL, IDI_APPLICATION);
    
    if (!RegisterClassEx(&wc)) {
        MessageBox(NULL, L"Window Registration Failed!", L"Error", MB_ICONEXCLAMATION | MB_OK);
        return 0;
    }
    
    // Create the window
    HWND hwnd = CreateWindowEx(
        0,                              // Optional window styles
        CLASS_NAME,                     // Window class
        L"My First Window",             // Window title
        WS_OVERLAPPEDWINDOW,           // Window style
        CW_USEDEFAULT, CW_USEDEFAULT,  // Position (x, y)
        800, 600,                      // Size (width, height)
        NULL,                          // Parent window
        NULL,                          // Menu
        hInstance,                     // Instance handle
        NULL                           // Additional application data
    );
    
    if (hwnd == NULL) {
        MessageBox(NULL, L"Window Creation Failed!", L"Error", MB_ICONEXCLAMATION | MB_OK);
        return 0;
    }
    
    ShowWindow(hwnd, nCmdShow);
    UpdateWindow(hwnd);
    
    // Message loop
    MSG msg = {};
    while (GetMessage(&msg, NULL, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }
    
    return 0;
}

// Window Procedure function
LRESULT CALLBACK WindowProc(HWND hwnd, UINT uMsg, WPARAM wParam, LPARAM lParam) {
    switch (uMsg) {
        case WM_DESTROY:
            PostQuitMessage(0);
            return 0;
        case WM_PAINT: {
            PAINTSTRUCT ps;
            HDC hdc = BeginPaint(hwnd, &ps);
            
            // Draw text
            TextOut(hdc, 50, 50, L"Hello, Windows GUI!", 19);
            
            EndPaint(hwnd, &ps);
            return 0;
        }
    }
    return DefWindowProc(hwnd, uMsg, wParam, lParam);
}

GUI Controls

Learn about the various GUI controls available in Windows programming and how to use them in your projects.

Understanding GUI Controls

GUI controls are the building blocks of any Windows application. They allow users to interact with your program through buttons, text boxes, checkboxes, and other interface elements. For your Matrix Operations and Encryption projects, you'll need several types of controls:

  • Buttons: For triggering actions like "Calculate", "Encrypt", "Decrypt"
  • Edit Controls: For inputting matrix values and text to encrypt/decrypt
  • Static Controls: For displaying labels and results
  • Combo Boxes: For selecting encryption algorithms
  • Group Boxes: For organizing related controls

Common GUI Controls

Control Types

Windows provides a variety of predefined control classes that you can use in your applications:

  • BUTTON: Creates a button control
  • EDIT: Creates an edit control for text input
  • STATIC: Creates a static control for text or images
  • COMBOBOX: Creates a combo box control
  • LISTBOX: Creates a list box control
  • SCROLLBAR: Creates a scroll bar control

// Define control IDs
#define ID_EDIT_INPUT    101
#define ID_BUTTON_CALC   102
#define ID_STATIC_RESULT 103

// Add these controls in WM_CREATE case of WindowProc
case WM_CREATE: {
    // Create input edit control
    HWND hEditInput = CreateWindowEx(
        0, L"EDIT", L"",
        WS_CHILD | WS_VISIBLE | WS_BORDER | ES_AUTOHSCROLL,
        50, 50, 200, 25,
        hwnd, (HMENU)ID_EDIT_INPUT, 
        ((LPCREATESTRUCT)lParam)->hInstance, NULL
    );
    
    // Create calculate button
    HWND hButtonCalc = CreateWindowEx(
        0, L"BUTTON", L"Calculate",
        WS_CHILD | WS_VISIBLE | BS_DEFPUSHBUTTON,
        50, 90, 100, 30,
        hwnd, (HMENU)ID_BUTTON_CALC,
        ((LPCREATESTRUCT)lParam)->hInstance, NULL
    );
    
    // Create result static control
    HWND hStaticResult = CreateWindowEx(
        0, L"STATIC", L"Result: ",
        WS_CHILD | WS_VISIBLE,
        50, 140, 200, 25,
        hwnd, (HMENU)ID_STATIC_RESULT,
        ((LPCREATESTRUCT)lParam)->hInstance, NULL
    );
    
    break;
}

// Handle button click in WM_COMMAND case
case WM_COMMAND: {
    switch (LOWORD(wParam)) {
        case ID_BUTTON_CALC: {
            // Get text from edit control
            wchar_t inputText[256];
            GetWindowText(GetDlgItem(hwnd, ID_EDIT_INPUT), inputText, 256);
            
            // Perform calculation (example: convert to integer and double)
            int value = _wtoi(inputText);
            int result = value * 2;
            
            // Display result
            wchar_t resultText[256];
            swprintf(resultText, 256, L"Result: %d", result);
            SetWindowText(GetDlgItem(hwnd, ID_STATIC_RESULT), resultText);
            
            break;
        }
    }
    break;
}

Event Handling

Learn how to handle user interactions and system events in your Windows applications.

Understanding Windows Messages

Windows applications are event-driven, meaning they respond to events triggered by user actions or system notifications. These events are communicated through messages sent to application's windows. Understanding how to handle these messages is crucial for creating interactive applications.

  • Messages: Notifications sent to a window when an event occurs
  • Message Loop: Retrieves messages from the queue and dispatches them
  • Window Procedure: Function that processes messages for a window
  • Message Handlers: Code that responds to specific messages

Message Processing Flow

How Messages Work

When a user interacts with your application, Windows generates messages and places them in a queue. Your application's message loop retrieves these messages and sends them to the appropriate window procedure for processing.

  • User performs action (clicks button, types text)
  • Windows generates a message
  • Message is placed in application queue
  • Message loop retrieves message
  • Window procedure processes message
  • Application responds to event

Essential Event Functions

Let's explore core functions and concepts for handling events:

Window Procedure (WindowProc)

An application-defined function that processes messages sent to a window.

Why Use It?

The Window Procedure is the heart of your application's event handling. It receives all messages for a window and determines how to respond to them.

For Matrix project: Handle button clicks for matrix operations and input validation.

For Encryption project: Handle button clicks for encryption/decryption and algorithm selection.

Parameters
  • hwnd: Handle to window
  • uMsg: Message identifier
  • wParam: Additional message information
  • lParam: Additional message information

GetMessage()

Retrieves a message from the calling thread's message queue.

Why Use It?

This function is the core of the message loop. It waits for a message to be placed in the queue and then retrieves it for processing.

Essential for both projects to maintain application responsiveness.

Parameters
  • lpMsg: Pointer to MSG structure to receive message
  • hWnd: Handle to window (NULL for any window)
  • wMsgFilterMin: Lowest message value to retrieve
  • wMsgFilterMax: Highest message value to retrieve

TranslateMessage()

Translates virtual-key messages into character messages.

Why Use It?

This function converts keyboard input (WM_KEYDOWN, WM_KEYUP) into character messages (WM_CHAR), making text input easier to handle.

For Matrix project: Handle numeric input for matrix values.

For Encryption project: Handle text input for encryption.

Parameters
  • lpMsg: Pointer to MSG structure containing message

DispatchMessage()

Dispatches a message to a window procedure.

Why Use It?

This function sends the message to the appropriate window procedure for processing. It's a crucial part of the message loop.

Essential for both projects to ensure messages are properly handled.

Parameters
  • lpMsg: Pointer to MSG structure containing message

                    return 0;

                case ID_COMBO_ALGORITHM: {

            PAINTSTRUCT ps;

            // Perform drawing operations
            // For Matrix project: Draw matrix grid
            // For Encryption project: Draw status indicators
            
            EndPaint(hwnd, &ps);
            return 0;
        }
        
        // Window destruction event
        case WM_DESTROY: {
            // Clean up resources
            PostQuitMessage(0);
            return 0;
        }
    }
    
    // Default message handling
    return DefWindowProc(hwnd, uMsg, wParam, lParam);
}

Graphics & Drawing

Learn how to create graphics and draw shapes, text, and images in your Windows applications.

Understanding GDI

The Graphics Device Interface (GDI) is the core of Windows graphics programming. It provides functions for drawing shapes, text, and images on the screen. For your projects, you'll use GDI to create visual elements and enhance the user interface.

  • Device Context (DC): A structure that defines graphic objects and their attributes
  • Graphic Objects: Pens, brushes, fonts, bitmaps, and palettes
  • Drawing Functions: Functions for drawing shapes, lines, and text
  • Mapping Modes: Ways to map logical coordinates to device coordinates

GDI Drawing Process

How GDI Works

When you need to draw in a window, you first obtain a device context, select appropriate graphic objects (like pens and brushes), and then use drawing functions to create graphics.

  • Begin painting (BeginPaint)
  • Get device context
  • Select graphic objects (pen, brush, font)
  • Perform drawing operations
  • Clean up objects
  • End painting (EndPaint)

Creating Graphics

Let's create a simple drawing application:

Graphics Drawing Code

// In WM_PAINT case of WindowProc
case WM_PAINT: {
    PAINTSTRUCT ps;
    HDC hdc = BeginPaint(hwnd, &ps);
    
    // Create a red pen for drawing
    HPEN hRedPen = CreatePen(PS_SOLID, 2, RGB(255, 0, 0));
    HPEN hOldPen = (HPEN)SelectObject(hdc, hRedPen);
    
    // Create a blue brush for filling
    HBRUSH hBlueBrush = CreateSolidBrush(RGB(0, 0, 255));
    HBRUSH hOldBrush = (HBRUSH)SelectObject(hdc, hBlueBrush);
    
    // Draw a rectangle
    Rectangle(hdc, 50, 50, 200, 150);
    
    // Select a green pen
    HPEN hGreenPen = CreatePen(PS_SOLID, 2, RGB(0, 255, 0));
    SelectObject(hdc, hGreenPen);
    
    // Draw an ellipse
    Ellipse(hdc, 250, 50, 400, 200);
    
    // Select a black pen for text
    HPEN hBlackPen = CreatePen(PS_SOLID, 1, RGB(0, 0, 0));
    SelectObject(hdc, hBlackPen);
    
    // Draw text
    TextOut(hdc, 50, 200, L"Graphics Demo", 13);
    
    // Draw text with formatting
    RECT textRect = {50, 230, 400, 260};
    DrawText(hdc, L"Formatted text drawing", -1, &textRect, DT_CENTER | DT_VCENTER);
    
    // Clean up objects
    SelectObject(hdc, hOldPen);
    SelectObject(hdc, hOldBrush);
    DeleteObject(hRedPen);
    DeleteObject(hGreenPen);
    DeleteObject(hBlackPen);
    DeleteObject(hBlueBrush);
    
    EndPaint(hwnd, &ps);
    return 0;
}

Understanding File Operations

File I/O (Input/Output) operations allow your applications to read data from and write data to files. For your projects, you'll use file operations to save and load matrix data and to read/write encrypted text files.

File I/O Process

How File Operations Work

File Operations Example

Let's create a simple file operations example:

File I/O Code

// Function to open a file dialog and read file content
void OpenFile(HWND hwnd) {
    OPENFILENAME ofn;
    wchar_t fileName[MAX_PATH] = L"";
    
    // Initialize OPENFILENAME structure
    ZeroMemory(&ofn, sizeof(ofn));
    ofn.lStructSize = sizeof(ofn);
    ofn.hwndOwner = hwnd;
    ofn.lpstrFilter = L"Text Files (*.txt)\0*.txt\0All Files (*.*)\0*.*\0";
    ofn.lpstrFile = fileName;
    ofn.nMaxFile = MAX_PATH;
    ofn.Flags = OFN_EXPLORER | OFN_FILEMUSTEXIST | OFN_HIDEREADONLY;
    ofn.lpstrDefExt = L"txt";
    
    // Display Open dialog
    if (GetOpenFileName(&ofn)) {
        // Open file for reading
        HANDLE hFile = CreateFile(
            fileName,           // File name
            GENERIC_READ,       // Open for reading
            FILE_SHARE_READ,    // Share for reading
            NULL,               // Default security
            OPEN_EXISTING,      // Existing file only
            FILE_ATTRIBUTE_NORMAL, // Normal file
            NULL                // No template
        );
        
        if (hFile != INVALID_HANDLE_VALUE) {
            // Read file content
            DWORD bytesRead;
            char buffer[1024];
            
            if (ReadFile(hFile, buffer, sizeof(buffer) - 1, &bytesRead, NULL)) {
                buffer[bytesRead] = '\0'; // Null-terminate
                
                // Convert to wide string for display
                wchar_t wideBuffer[1024];
                MultiByteToWideChar(CP_ACP, 0, buffer, -1, wideBuffer, 1024);
                
                // Display content in edit control
                SetWindowText(GetDlgItem(hwnd, ID_EDIT_CONTENT), wideBuffer);
            }
            
            // Close file
            CloseHandle(hFile);
        }
    }
}

// Function to save content to file
void SaveFile(HWND hwnd) {
    OPENFILENAME ofn;
    wchar_t fileName[MAX_PATH] = L"";
    
    // Initialize OPENFILENAME structure
    ZeroMemory(&ofn, sizeof(ofn));
    ofn.lStructSize = sizeof(ofn);
    ofn.hwndOwner = hwnd;
    ofn.lpstrFilter = L"Text Files (*.txt)\0*.txt\0All Files (*.*)\0*.*\0";
    ofn.lpstrFile = fileName;
    ofn.nMaxFile = MAX_PATH;
    ofn.Flags = OFN_EXPLORER | OFN_PATHMUSTEXIST | OFN_HIDEREADONLY | OFN_OVERWRITEPROMPT;
    ofn.lpstrDefExt = L"txt";
    
    // Display Save dialog
    if (GetSaveFileName(&ofn)) {
        // Get content from edit control
        wchar_t content[4096];
        GetWindowText(GetDlgItem(hwnd, ID_EDIT_CONTENT), content, 4096);
        
        // Convert to multibyte for file writing
        char buffer[4096];
        WideCharToMultiByte(CP_ACP, 0, content, -1, buffer, 4096, NULL, NULL);
        
        // Open file for writing
        HANDLE hFile = CreateFile(
            fileName,           // File name
            GENERIC_WRITE,      // Open for writing
            0,                  // Do not share
            NULL,               // Default security
            CREATE_ALWAYS,      // Create or overwrite
            FILE_ATTRIBUTE_NORMAL, // Normal file
            NULL                // No template
        );
        
        if (hFile != INVALID_HANDLE_VALUE) {
            // Write content to file
            DWORD bytesWritten;
            WriteFile(hFile, buffer, strlen(buffer), &bytesWritten, NULL);
            
            // Close file
            CloseHandle(hFile);
            
            MessageBox(hwnd, L"File saved successfully!", L"Success", MB_OK | MB_ICONINFORMATION);
        }
    }
}

// Caesar cipher implementation
void CaesarEncrypt(const wchar_t* plaintext, const wchar_t* key, wchar_t* ciphertext) {
    int shift = _wtoi(key) % 26;
    int i = 0;
    while (plaintext[i] != L'\0') {
        if (plaintext[i] >= L'A' && plaintext[i] <= L'Z') {
            ciphertext[i] = L'A' + (plaintext[i] - L'A' + shift) % 26;
        } else if (plaintext[i] >= L'a' && plaintext[i] <= L'z') {
            ciphertext[i] = L'a' + (plaintext[i] - L'a' + shift) % 26;
        } else {
            ciphertext[i] = plaintext[i];
        }
        i++;
    }
    ciphertext[i] = L'\0';
}

// VigenÃ¨re cipher implementation
void VigenereEncrypt(const wchar_t* plaintext, const wchar_t* key, wchar_t* ciphertext) {
    int keyLen = wcslen(key);
    int i = 0, j = 0;
    while (plaintext[i] != L'\0') {
        if (plaintext[i] >= L'A' && plaintext[i] <= L'Z') {
            int shift = toupper(key[j % keyLen]) - L'A';
            ciphertext[i] = L'A' + (plaintext[i] - L'A' + shift) % 26;
            j++;
        } else if (plaintext[i] >= L'a' && plaintext[i] <= L'z') {
            int shift = tolower(key[j % keyLen]) - L'a';
            ciphertext[i] = L'a' + (plaintext[i] - L'a' + shift) % 26;
            j++;
        } else {
            ciphertext[i] = plaintext[i];
        }
        i++;
    }
    ciphertext[i] = L'\0';
}
